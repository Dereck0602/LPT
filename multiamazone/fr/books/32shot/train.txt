1	Un livre à lire au moins une fois ! Contrairement à sa description, ce livre s'adresse à la fois aux lecteurs avertis qui sont à la pointe de la technologie et qui recherchent se qui se cache sous le mot magique "AJAX", mais également aux débutants qui ont des notions d'HTML et qui veulent découvrir comment développer un site internet de façon fluide et conviviale pour leurs visiteurs. Le javascript est présenté dans sa globalité avec les mots-clés, les instructions, les fonctions, les principaux objets, etc. L'auteur nous fait découvrir comment utiliser au mieux le javascript et le XML afin de rendre nos applications web beaucoup plus rapides. Les débutants pourront ainsi apprendre les rudiments du javascript avant de se lancer dans l'utilisation d'AJAX, alors que les initiés pourront découvrir ce qui se cache derrière le terme "AJAX". Les points positifs : * L'auteur s'attarde sur les technologies regroupées par AJAX, ce qui permet de tout comprendre : avant d'étudier les requêtes asynchrones client/serveur, il faut forcément connaître le javascript et le XML. * La lecture du livre est plaisante avec des chapitres qui s'enchaînent admirablement bien. * Les exemples sont clairs et bien trouvés : les exemples portent sur des points précis du langage, et des exemples plus concrets sont donnés à la fin du livre. Le code fournit est assez rigoureux et documenté. Les points négatifs : * La présentation de javascript n'évoque pas les aspects avancés de javascript tels que la programmation d'objets, l'utilisation de pointeur de fonction, le traîtement des événements, etc. * Le code des exemples sur AJAX est assez lourd à lire : par exemple, l'auteur nous répète à chaque exemple l'instanciation de l'objet XMLHttpRequest avec le traîtement de la réponse du serveur, puis nous donne le code XHTML de la page avec le code javascript (un doublon), puis le code XHTML de la page avec du css pour la présentation (de nouveau un doublon). C'est pratique si on ne lit qu'un seul exemple, mais lourd si on lit les chapitres de bout en bout. * Ce livre ne présente que l'AJAX, ca permet de ne pas s'égarer, mais ne nous présente que l'aspect client de l'AJAX, c'est-à-dire : uniquement la partie javascript. Il aurait été judicieux je pense de nous présenter au moins un petit exemple en PHP ou Java ou .NET d'un traîtement de requête côté serveur avec l'encodage du fichier XML de retour. Ce livre est à lire au moins une fois, et est une référence à garder près de soi lors de nos développement AJAX !
